# Units

## 核心理解

在计算机和数据系统语境里，大家经常把两类完全不同的“单位”混在一起：

- 一类是存储单位，描述数据占了多少空间，例如 `KB`、`MB`、`GB`、`TB`。
- 一类是数量级单位，描述对象有多少个，例如 `k`、`M`、`B`、`T`。

两者看起来都像“字母缩写”，但底层含义不同：

- 存储单位通常以字节 `B` 为基础，在工程语境里常按二进制换算。
- 数量级单位通常以十进制换算，用来表示 token 数、参数量、请求量、样本量等规模。

如果不先区分这两类单位，就很容易误读诸如 `70B parameters`、`15T tokens`、`32GB memory` 这类表达。

## 两类单位的区别

### 存储单位

存储单位回答的是“这份数据占多少空间”。它关注的是容量、传输体积和内存/磁盘消耗。

| **单位** | **缩写** | **数量级（以字节为单位）** | **应用场景** |
| --- | --- | --- | --- |
| 位（比特） | bit | 1 bit = 0.125 B | 计算机底层逻辑传输，如网络带宽 `Mbps`、编码与压缩的最小信息单位。 |
| 字节 | B | 1 B = 8 bit | 基本存储单位，用于表示字符、文件大小和内存地址粒度。 |
| 千字节 | KB | 1 KB = 2^10 B = 1024 B | 小型文件、简单配置文件、缩略图、缓存块大小。 |
| 兆字节 | MB | 1 MB = 2^20 B = 1024 KB | 音频、短视频片段、安装包、中等规模数据文件。 |
| 吉字节 | GB | 1 GB = 2^30 B = 1024 MB | 高清视频、大型模型权重分片、磁盘和显存容量。 |
| 太字节 | TB | 1 TB = 2^40 B = 1024 GB | 数据集存储、对象存储、NAS、企业级服务器容量。 |

### 数量单位

数量单位回答的是“有多少个对象”。它关注的是计数，而不是这些对象占了多少字节。

| **单位** | **缩写** | **数量级（以 1 为基准）** | **应用场景 / 说明** |
| --- | --- | --- | --- |
| 千 | k | 10^3 = 1000 | 例如 `10k samples`、`100k users`、`8k context`。 |
| 兆（百万） | M | 10^6 = 1,000,000 | 常见于样本量、参数量、请求量，例如 `7M parameters`。 |
| 十亿（billion） | B | 10^9 = 1,000,000,000 | LLM 里最常见，例如 `7B`、`70B parameters`。这里的 `B` 是 billion，不是 byte。 |
| 万亿（trillion） | T | 10^12 = 1,000,000,000,000 | 常见于训练 token 总量、计算规模，例如 `15T tokens`。 |

## 为什么容易混淆

最常见的混淆来自字母 `B`：

- 在存储单位里，`B` 表示 `Byte`，即字节。
- 在数量单位里，`B` 常表示 `Billion`，即十亿。

所以：

- `32GB` 通常表示 32 吉字节容量。
- `70B parameters` 通常表示 700 亿参数，而不是 70 字节。

另一个常见混淆是二进制和十进制：

- 存储容量在很多工程上下文里常按 `1024` 进位。
- 数量级缩写几乎都按 `1000` 进位。

因此：

- `1 MB` 在这里按 1024 × 1024 字节理解。
- `1M tokens` 则是 1,000,000 个 token，而不是 1,048,576 个 token。

## 在 AI / LLM 中如何读这些数字

在模型、训练和推理语境中，常见表达可以这样理解：

- `7B model`：大约 70 亿参数的模型。
- `70B parameters`：700 亿参数。
- `15T tokens`：训练或处理了 15 万亿个 token。
- `24GB VRAM`：显卡有 24GB 显存。
- `500MB checkpoint`：模型文件大小约 500MB。

这里要注意，参数量和 token 数是“数量单位”，模型文件、显存、磁盘占用是“存储单位”。它们相关，但不等价。

例如：

- 一个 `7B` 模型不等于只占 `7GB`。
- 模型最终占用多少存储空间，还取决于参数精度，例如 FP32、FP16、INT8、INT4。

## 一个直观例子

假设某模型有 `7B parameters`：

- 如果每个参数使用 FP16 存储，大约每个参数占 2 字节。
- 仅参数本体理论上大约需要 `14GB` 空间。
- 实际部署时还可能叠加 optimizer state、KV cache、激活值和框架额外开销。

这说明：

- `7B` 说的是“有多少个参数”。
- `14GB` 说的是“这些参数大约占多少空间”。

两者不能混为一谈。

## 相关提醒

在更严格的标准里，二进制存储单位也会写成：

- `KiB` = 1024 B
- `MiB` = 1024 KiB
- `GiB` = 1024 MiB
- `TiB` = 1024 GiB

不过在很多产品文案、工程讨论和日常表达里，人们仍然会直接写 `KB`、`MB`、`GB`、`TB`。阅读时要结合上下文判断。

## 与相关节点的关系

- Computer Science：这是更底层的通用单位知识，适用于存储、网络、系统和数据处理。
- LLM：模型参数量、上下文长度和训练 token 数会频繁使用这里的数量级表示。
- Token：`k`、`M`、`T` 在 token 数量表达中极其常见，后续可单独展开。
