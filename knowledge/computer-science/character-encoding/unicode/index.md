# Unicode编码

## 核心理解

Unicode 是国际标准字符集，它试图为世界各种语言中的字符分配统一的唯一编号，使不同语言、符号和书写系统能够在同一个字符体系下表示。

如果说 ASCII 主要解决的是英文字符的编号问题，那么 Unicode 解决的是“全球字符如何放进同一套标准体系”这个问题。

Unicode 的编码范围是 `0x0000 - 0x10FFFF`。这个范围覆盖了一百多万个可能的码点位置，用来容纳现代文本系统所需的大量字符与符号。

例如：

- 汉字 `中` 的码点是 `U+4E2D`
- 大写字母 `A` 的码点是 `U+0041`

这里的 `U+` 表示 Unicode 码点写法，后面的数值通常使用十六进制表示。

因此，Unicode 首先是一套统一字符标准。它回答的是“字符是什么、对应哪个标准编号”，而不是“这个字符最终以哪几个字节存进文件”。

## 为什么需要 Unicode

ASCII 只覆盖 128 个字符，主要面向英文环境。

这在早期已经够用，但随着计算机需要处理全球语言，问题很快出现：

- 中文无法完整表示
- 日文、韩文无法统一表示
- 大量变音字符和特殊符号无法纳入同一体系
- 数学符号、货币符号、emoji 等也需要统一标准

如果每种语言都依赖彼此不兼容的本地编码，那么文本跨系统交换时就很容易出现乱码或误解码。

Unicode 的目标就是尽量把全球主要字符统一纳入同一个标准字符集。

## 其它编码

随着计算机的普及，在不同地区和国家又出现了很多字符编码，例如：

- 大陆的 `GB2312`
- 港台常见的 `Big5`
- 日本的 `Shift JIS`

这些编码通常服务于特定语言环境，能在局部场景中解决字符表示问题，但彼此之间并不天然兼容。这也是 Unicode 最终变得重要的原因之一。

## 码点

码点（code point）可以理解为 Unicode 给某个字符分配的标准编号。

它的关键含义是：

- 它标识“这个字符是谁”
- 它属于抽象字符层
- 它不是最终存储时的字节序列

例如：

- `A` 的码点是 `U+0041`
- `中` 的码点是 `U+4E2D`
- `😀` 的码点是 `U+1F600`

所以，码点是字符的标准编号，不等于它在文件或内存里的最终二进制表示。

## Unicode 与 UTF-8、UTF-16、UTF-32 的关系

Unicode 和 UTF-8、UTF-16、UTF-32 不是同一层概念。

更准确地说：

- Unicode 定义字符和码点
- UTF-8、UTF-16、UTF-32 定义码点如何转换为字节序列

也就是说：

- Unicode 更像统一字符表
- UTF 则是 Unicode Transformation Format，即 Unicode 转换格式
- 后面的数字表示这种格式至少以多少 bit 为基本编码单元

例如：

- UTF-8 以 8 bit，也就是 1 字节为最小编码单元
- UTF-16 以 16 bit，也就是 2 字节为最小编码单元
- UTF-32 以 32 bit，也就是 4 字节为最小编码单元

需要注意，不能把“Unicode 范围大，所以需要 1 到 3 个字节表示”这样理解。正确说法应当是：

- 同一个 Unicode 码点在不同 UTF 格式下会有不同的字节表示
- UTF-8 使用 `1` 到 `4` 个字节
- UTF-16 使用 `2` 或 `4` 个字节
- UTF-32 固定使用 `4` 个字节

## Unicode 码点的存储方式

常见的 Unicode 存储方式有三种：

| 编码方式 | 编码类型 | 字节长度范围 | 编码逻辑及特点 |
| --- | --- | --- | --- |
| UTF-8 | 变长字符编码 | 1 至 4 个字节 | 根据码点大小决定字节数，兼容 ASCII，是最常用的 Unicode 编码方式 |
| UTF-16 | 变长字符编码 | 2 字节或 4 字节 | 多数常用字符使用 2 字节，部分较高码点字符使用 4 字节 |
| UTF-32 | 固定长度编码 | 始终 4 个字节 | 每个码点直接使用 4 字节表示，结构简单，但空间占用大 |

UTF 的意义就在于：同样是 Unicode 码点，可以按不同的存储和传输格式编码成字节序列，以在兼容性、空间占用和处理复杂度之间做取舍。

## UTF-8 为什么最常用

UTF-8 最常见，主要因为它同时满足几个重要条件：

- 对 ASCII 完全兼容
- 英文文本空间效率高
- 可以覆盖全部 Unicode 字符
- 在网络协议、文件系统、编程语言和数据库中广泛采用

这使它成为现代文本系统中的默认选择之一。

## UTF-8 的编码规则

UTF-8 是变长编码。一个字符最终使用多少字节，取决于它的 Unicode 码点范围。

### 单字节字符

适用码点范围：

- `U+0000` 到 `U+007F`

字节格式：

```text
0xxxxxxx
```

这 7 位直接携带码点值，因此这一段和 ASCII 完全兼容。

例如：

- `U+0041`（`A`）的 UTF-8 编码是 `0x41`
- `U+0021`（`!`）的 UTF-8 编码是 `0x21`

### 双字节字符

适用码点范围：

- `U+0080` 到 `U+07FF`

字节格式：

```text
110xxxxx
10xxxxxx
```

含义是：

- 首字节去掉前缀 `110` 后保留 5 位
- 第二字节去掉前缀 `10` 后保留 6 位
- 两部分拼起来形成码点二进制值

例如：

- `U+00E9`（`é`）的 UTF-8 编码是 `C3 A9`

### 三字节字符

适用码点范围：

- `U+0800` 到 `U+FFFF`

字节格式：

```text
1110xxxx
10xxxxxx
10xxxxxx
```

例如：

- `U+4E2D`（`中`）的 UTF-8 编码是 `E4 B8 AD`

### 四字节字符

适用码点范围：

- `U+10000` 到 `U+10FFFF`

字节格式：

```text
11110xxx
10xxxxxx
10xxxxxx
10xxxxxx
```

例如：

- `U+1F469`（`👩`）的 UTF-8 编码是 `F0 9F 91 A9`

## 与 ASCII 的关系

Unicode 不是完全替代 ASCII，而是把 ASCII 覆盖的字符纳入更大的统一体系。

例如：

- `A` 在 ASCII 中是十进制 `65`
- `A` 在 Unicode 中的码点是 `U+0041`

在 UTF-8 中，ASCII 范围内字符仍然保持单字节表示，且字节值与 ASCII 一致。

因此可以说：

- ASCII 是早期英文字符集
- Unicode 是更完整的全球字符标准
- UTF-8 通过兼容 ASCII，平滑承接了旧系统和现代系统之间的文本表示

## 在系统中的实际位置

为了避免把“步骤”和“产物”混在一起，可以分层理解：

### 字符标准层

Unicode 定义字符和码点。

例如：

- 字符 `中`
- 码点 `U+4E2D`

这一层说明字符在标准体系中是谁。

### 存储与传输层

UTF-8、UTF-16、UTF-32 把码点转换成字节序列，用于：

- 文件存储
- 网络传输
- 内存中的文本表示

例如，字符 `中` 在 UTF-8 中会编码成 `E4 B8 AD`。

### 上层文本处理

程序拿到文本之后，后续还可以继续做分词、搜索、排序、渲染或模型处理。

如果输入源是文件或网络字节流，程序通常要先按 UTF-8 等规则解码成字符序列；如果输入本来就是运行时字符串，那么这一步已经在更底层完成，不需要再显式写一次“解码回字符”。

## 常见误区

### 误区一：Unicode 就是 UTF-8

不对。

- Unicode 是字符标准
- UTF-8 是 Unicode 的一种编码实现

### 误区二：码点就是字节序列

不对。

- 码点是字符编号
- 字节序列是码点经过 UTF 编码后的存储结果

### 误区三：所有 Unicode 字符都用 3 个字节存储

不对。

- UTF-8 使用 `1-4` 字节
- UTF-16 使用 `2` 或 `4` 字节
- UTF-32 固定使用 `4` 字节

### 误区四：Unicode 和 ASCII 完全无关

不对。

- Unicode 保留并扩展了 ASCII 覆盖的字符
- UTF-8 还保持了对 ASCII 的兼容性

## 与相关节点的关系

- Character Encoding：Unicode 属于字符编码知识体系中的核心基础节点。
- ASCII编码：ASCII 是早期英文字符集，Unicode 扩展并统一了更广泛的字符表示。
- UTF-8：UTF-8 是 Unicode 最常用的编码实现方式。
- 乱码：乱码常见原因是编码和解码规则不一致。
