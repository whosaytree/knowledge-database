# ASCII编码

## 核心理解

ASCII（American Standard Code for Information Interchange）是一套早期字符编码标准，用固定的数字编号表示英文字母、数字、标点和若干控制字符。

ASCII 编码一共定义了 128 个字符的编码规则，使用 7 位二进制表示，范围是 `0x00` 到 `0x7F`。这些字符组成的集合就叫做 ASCII 字符集。

其中，`0x00 - 0x7F` 是用十六进制表示的 ASCII 编码范围，用来直观展示 ASCII 字符集覆盖的数值区间。十六进制数通常以 `0x` 开头。

它的核心特点是：

- 只覆盖有限字符集，主要面向英文环境。
- 每个字符对应一个确定的整数编码值。
- 标准 ASCII 使用 7 bit，最多表示 128 个字符，编码范围是 `0` 到 `127`。

例如：

- `A` 对应十进制 `65`
- `a` 对应十进制 `97`
- `0` 对应十进制 `48`

因此，ASCII 的本质是“字符到数值编号的映射规则”。

## 编码内容

ASCII 大致分为两类内容：

### 控制字符

编码值 `0` 到 `31` 以及 `127` 多数属于控制字符，不直接显示为普通文本字符，而用于控制设备或文本流行为。

常见例子：

- `LF`：换行
- `CR`：回车
- `TAB`：制表
- `ESC`：转义

这些字符在终端、文本协议和老式设备通信里很常见。

### 可打印字符

编码值 `32` 到 `126` 是常见可显示字符，包括：

- 空格和标点
- 数字 `0-9`
- 大写字母 `A-Z`
- 小写字母 `a-z`

例如：

- `32` 是空格
- `33` 是 `!`
- `65` 是 `A`
- `90` 是 `Z`
- `97` 是 `a`
- `122` 是 `z`

## 工作机制

ASCII 编码的工作方式并不复杂：

1. 先定义一张字符表。
2. 表中每个字符绑定一个唯一编号。
3. 计算机在存储和传输文本时，实际处理的是这些编号，而不是字符外形本身。
4. 显示设备或程序再根据编号把字符渲染出来。

例如，字符串 `ABC` 在 ASCII 里可表示为：

- `A` -> `65`
- `B` -> `66`
- `C` -> `67`

如果按二进制写，就是这些十进制数对应的二进制形式。

## 为什么只需要 7 bit

标准 ASCII 一共定义了 128 个符号，而 `2^7 = 128`，所以 7 bit 就足够覆盖全部编码值。

这意味着：

- 最早的 ASCII 标准理论上不需要完整的 8 bit 字节。
- 但现代计算机通常仍然用 1 byte 来存放一个 ASCII 字符，只是最高位为 `0`。

因此在现代系统里，人们常说“ASCII 字符占 1 个字节”，这是工程实现上的常见表达。

## 局限性

ASCII 的主要局限是字符集太小。

它可以很好表示：

- 英文大小写字母
- 阿拉伯数字
- 基础标点
- 少量控制字符

但它不能直接表示：

- 中文
- 日文
- 韩文
- 大量欧洲语言变音字符
- emoji

这也是后来出现更大字符集和更通用编码体系的原因，例如 Unicode 及其常见实现 UTF-8。

## ASCII、Tokenizer 与 Embedding 的区别

这三者都和“文本表示”有关，但处在完全不同的层次。

### ASCII 编码

ASCII 是字符编码，用来规定字符如何表示为数值和二进制。

它解决的是：

- 文本怎么存储
- 文本怎么传输
- 程序怎么读取字符

例如：

- `A -> 65`
- `a -> 97`

它是离散、固定、人工定义好的映射规则，不表达语义相似性。

### Tokenizer

Tokenizer 不负责把字符存成字节，而是负责把文本切成模型可处理的 token，并映射成 token id。

例如：

`"hello world"` 可能被切成：

- `["hello", " world"]`

再映射成：

- `[15339, 1917]`

它解决的是：

- 文本如何进入模型
- 文本如何变成离散 token 序列

所以，tokenizer 和字符编码不是同一层。字符编码决定文本如何进入计算机，tokenizer 决定文本如何进入模型。

### Embedding

Embedding 会把 token id 进一步变成连续向量表示。

例如：

- `15339 -> [0.12, -0.37, 0.91, ...]`

它解决的是：

- 离散符号如何变成神经网络可计算的连续输入
- 文本片段如何在向量空间里表达语义关系

ASCII 中 `A` 和 `B` 的编码值接近，并不代表它们语义接近；embedding 中两个向量距离接近，才通常表示语义更相似。

## 在大模型中分别什么时候使用

在大模型系统里，这几层通常是依次发生的：

```text
文本
-> 字符编码（如 UTF-8 / ASCII）
-> tokenizer
-> token ids
-> embedding
-> Transformer
```

这意味着：

- 字符编码主要用于文本的存储、读取和传输。
- tokenizer 用于把文本切成 token 并编号。
- embedding 用于把 token 的离散编号变成可参与神经网络计算的连续向量表示。

因此，大模型不是“用编码还是用 embedding”二选一，而是先经过编码和 token 化，再进入 embedding。

## 在现代系统中的地位

虽然 ASCII 本身很早，但它今天仍然非常重要，因为很多现代编码都和 ASCII 兼容。

例如 UTF-8 的一个关键性质是：

- 对于 ASCII 范围内的字符，UTF-8 编码结果和 ASCII 完全一致。

这带来两个直接好处：

- 英文文本和大量程序源代码可以自然兼容旧系统。
- 许多协议、文件格式和编程语言都默认把 ASCII 区域当作基础字符集合。

所以 ASCII 虽然不再足够覆盖现代多语言文本，但仍然是很多文本系统的基础层。

## 与相关节点的关系

- Computer Science：ASCII 是字符表示和文本存储的基础知识。
- Units：ASCII 字符通常按字节存储，和 bit、byte 的概念直接相关。
- Tokenizer：Tokenizer 负责把文本切成模型可处理的 token 并映射为 token id。
- Embedding：Embedding 负责把 token id 变成连续向量表示。
- Unicode / UTF-8：它们扩展了 ASCII 无法覆盖的多语言字符表示能力。
