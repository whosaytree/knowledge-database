# Optimization

## 核心理解

Optimization 讨论的是：模型参数如何根据损失函数和梯度信息被实际更新。

在机器学习里，模型结构决定“能表示什么”，损失函数决定“想优化什么”，而优化器决定“参数具体怎么走”。很多训练稳定性、收敛速度和最终效果差异，都会体现在这一层。

## 关键问题

- 梯度方向如何估计和利用？
- 更新步长如何控制？
- 历史梯度信息是否要保留？
- 不同参数是否应使用不同的有效学习率？
- 如何在收敛速度、稳定性、显存开销和泛化能力之间取舍？

## 主要方向

- [Backpropagation](backpropagation/index.md)：在可导模型中高效计算各参数梯度的核心机制。
- [SGD](sgd/index.md)：最基础的随机梯度下降方法。
- [Adam](adam/index.md)：结合动量与自适应学习率的常用优化器。
- [No Free Lunch Theorem](no-free-lunch-theorem/index.md)：说明为什么不存在对所有优化问题都普适最优的算法。
- Momentum：在 SGD 上加入梯度历史平滑。
- RMSProp：使用梯度平方历史调节更新步长。
- AdamW：把 weight decay 与 Adam 更新解耦。

## 与机器学习的关系

Optimization 是模型训练过程中的核心组成部分。

同一个模型结构、同一份数据、同一套损失函数，在不同优化器和学习率调度策略下，训练曲线和最终效果都可能明显不同。因此它既是理论问题，也是强工程问题。

## 常见优化器比较

在实践中，优化器选择通常取决于当前模型、数据规模、训练预算和泛化目标。

在当前常见方法里，SGD 和 Adam 是最常见的一组对照。

### SGD 与 Adam 的关键区别

| 维度 | SGD | Adam |
| --- | --- | --- |
| 学习率特性 | 通常使用统一的全局学习率，并依赖手动调整或调度策略 | 为不同参数提供自适应步长，更新幅度会受历史梯度统计影响 |
| 动量机制 | 纯 SGD 不包含动量；实践中常配合 Momentum 使用 | 内置一阶矩估计，天然包含类似动量的效果 |
| 收敛表现 | 训练前期通常较慢，路径抖动更明显 | 训练前期通常下降更快，更新更平滑 |
| 超参数敏感性 | 对学习率和调度策略更敏感 | 通常比纯 SGD 更容易起步，但仍然依赖合适超参数 |
| 内存需求 | 低，主要保存参数与当前梯度 | 更高，需要额外维护一阶矩和二阶矩状态 |
| 最终效果 | 某些任务中最终泛化可能更强 | 某些任务中更容易先得到较好的训练结果，但最终泛化不一定占优 |

### SGD 的特点

优点：

- 规则简单，额外状态少，显存和存储开销低。
- 学习机制直接，适合作为理解其他优化器的基础。
- 在一些任务中，精心调优后的 SGD 可能获得更好的最终泛化效果。

局限：

- 对学习率和学习率调度更敏感。
- 训练前期通常更慢，路径抖动更明显。
- 缺少按参数自适应调整步长的能力。

### Adam 的特点

优点：

- 训练前期通常下降更快。
- 对不同参数尺度的适应性更强。
- 在复杂模型和大规模训练中通常更容易先跑起来。

局限：

- 需要维护额外优化器状态，显存和 checkpoint 开销更高。
- 默认配置虽然常常可用，但并不意味着不需要调参。
- 在一些任务中，最终泛化表现可能不如精心调优的 SGD。

### 如何选择

一般可以这样粗略理解：

- 如果更关心训练是否容易启动、是否能较快得到可用结果，Adam 往往更常见。
- 如果训练配方已经成熟，并且更关注最终泛化表现，可以尝试 SGD 或 SGD with Momentum。
- 在很多实际系统里，优化器选择往往要和 learning rate schedule、weight decay、batch size 一起看，而不能孤立比较。
