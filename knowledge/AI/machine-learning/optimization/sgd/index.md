# SGD

## 核心理解

SGD（Stochastic Gradient Descent，随机梯度下降）是最基础、也最重要的一类参数优化方法。

它的核心思想非常直接：先定义一个损失函数来衡量模型预测和真实目标之间的误差，然后计算当前参数下损失函数对参数的梯度，再沿着梯度的反方向更新参数，使损失逐步下降。

如果把模型训练理解为“在一个高维地形中往低处走”，那么 SGD 就是在每一步根据当前位置的局部坡度，决定下一步往哪里走、走多远。

虽然很多现代优化器都比它更复杂，但 SGD 仍然是理解神经网络训练机制的起点。许多后续方法，包括 Momentum、RMSProp、Adam，本质上都可以看作是对 SGD 更新规则的改造。

## 为什么叫 Stochastic

最理想的梯度下降，通常会对整个训练集计算一次完整梯度，然后再更新参数。这种方法可以叫作 batch gradient descent。

如果训练集很大，这样做会有两个问题：

- 每一步更新都非常慢。
- 需要先扫完整个数据集，参数才能动一次。

SGD 的关键变化是：不再等看完整个数据集才更新，而是只用一个样本，或者更常见地，用一个 mini-batch，先估计当前梯度，再立刻更新参数。

因此这里的 stochastic，指的是梯度不再是“全数据上的精确梯度”，而是“基于随机抽样样本得到的近似梯度”。

这会带来两个直接结果：

- 梯度带噪声，不会每一步都精确指向全局平均下降方向。
- 但每一步更便宜，整体训练效率更高，也更适合大规模数据和神经网络训练。

在现代深度学习语境里，大家说 SGD，很多时候实际指的是 mini-batch SGD，而不是“单样本更新”的最狭义形式。

## 工作机制 / 构建过程

设参数为 $\theta_t$，第 $t$ 步时根据当前 mini-batch 计算得到梯度 $g_t$。

最基本的 SGD 更新规则是：

$$
\theta_{t+1} = \theta_t - \eta g_t
$$

其中：

- $\theta_t$ 是当前参数。
- $g_t$ 是当前梯度。
- $\eta$ 是学习率，决定这一步走多远。

这条规则非常朴素：

- 梯度告诉我们损失增长最快的方向。
- 所以沿梯度反方向走，通常会让损失下降。
- 学习率控制更新幅度，太大容易震荡，太小又会收敛过慢。

如果把它展开成训练流程，通常是：

1. 取一个 mini-batch 数据。
2. 前向计算得到模型输出与损失。
3. 反向传播计算每个参数的梯度。
4. 按照 SGD 规则更新参数。
5. 进入下一个 mini-batch，重复上述过程。

因此，SGD 的本质并不复杂。它是“梯度告诉方向，学习率控制步长”的直接实现。

## SGD 为什么能工作

SGD 的每一步只使用当前 mini-batch 的梯度，所以单步方向可能带噪声，不一定等于整个训练集上的真实下降方向。

但如果数据采样是随机的，那么从整体上看，这些 noisy gradient 往往仍然能提供有用的下降趋势。虽然每一步都可能有抖动，但在足够多步之后，参数通常仍然会朝着较优区域移动。

这种“带噪声但成本低”的更新方式，恰好适合现代机器学习：

- 模型参数很多。
- 数据量很大。
- 不可能每次都精确计算全量梯度。
- 更关心整体训练效率，而不是单步绝对精确。

所以 SGD 的成功，不在于每一步都特别聪明，而在于它足够简单、便宜、稳定，并且在大规模迭代下有效。

## 学习率为什么这么重要

在 SGD 里，学习率往往是最关键的超参数。

更新公式里真正控制步长的就是 $\eta$：

$$
\theta_{t+1} = \theta_t - \eta g_t
$$

如果学习率太大：

- 参数可能在最优点附近来回震荡。
- 训练损失可能不降反升。
- 数值可能直接发散。

如果学习率太小：

- 每一步更新太弱。
- 收敛非常慢。
- 容易长时间停留在效果不理想的区域。

因此，SGD 往往需要配合学习率调度策略使用，例如：

- 训练过程中逐步降低学习率。
- warmup 后再进入正常衰减。
- 在不同阶段采用不同学习率规模。

很多时候，SGD 是否好用，不只是由优化器名字决定，而是由“SGD + learning rate schedule”的整体配置决定。

## SGD 的优点和局限

SGD 的优点包括：

- 形式简单，容易理解。
- 额外状态少，显存和存储开销低。
- 在很多任务中具有良好的泛化表现。
- 是很多优化器变体的基础参照。

它的局限也很明显：

- 对学习率更敏感。
- 梯度噪声较大，训练曲线常有抖动。
- 不同参数共享同一个全局学习率，缺少自适应能力。
- 在病态曲率、尺度差异大的参数空间里，优化效率可能不高。

也正因为这些局限，后续才发展出 Momentum、RMSProp、Adam 等改进方案。

## Mini-batch、Batch、Stochastic 的关系

这几个概念很容易混淆，需要明确区分。

Batch Gradient Descent 指的是：

- 每次都用整个训练集计算一次梯度，再更新参数。

Stochastic Gradient Descent 在最严格定义里指的是：

- 每次只用一个样本计算梯度，然后更新参数。

而现代深度学习里更常见的是 Mini-batch SGD：

- 每次用一个小批量样本估计梯度，再更新参数。

因此今天工程语境里说 SGD，往往默认包含 mini-batch 训练方式。严格术语上它不完全等于“单样本随机梯度下降”，但在实践中大家通常沿用 SGD 这个名字。

## SGD with Momentum 是什么

纯 SGD 只看当前一步梯度，因此更新方向容易受局部噪声影响。

Momentum 在 SGD 基础上加入“历史方向累积”，让参数更新更稳定，也更容易沿着长期一致的方向持续前进。它通常可以写成另一类节点，而不应和纯 SGD 混为一谈。

从知识结构上看：

- SGD 是最基础母体。
- Momentum 是基于 SGD 的直接增强。
- Adam 则进一步把动量和自适应步长结合起来。

所以在这个知识库里，SGD 节点应先讲清楚最基础更新规则，再把 Momentum、Adam 作为后续分化路线去关联。

## 实现方法与工程细节

SGD 的工程实现相对简单，因为它几乎不需要复杂额外状态。

最基础版本中，每个参数只需要：

- 参数值本身
- 当前梯度

这意味着和 Adam 相比，SGD 的优化器状态显存开销更小。Adam 需要为每个参数额外维护一阶矩和二阶矩，而纯 SGD 不需要。

实际工程中，SGD 常见配置项包括：

- `lr`：学习率。
- `momentum`：是否加入动量，以及动量系数大小。
- `weight_decay`：权重衰减。
- `nesterov`：是否使用 Nesterov Momentum。
- `dampening`：动量项的额外抑制设置。

不过如果配置了 `momentum`，严格来说使用的就已经不是最朴素的 SGD，而是带动量的 SGD 变体。

## 与相关节点的关系

- Optimization：SGD 是 Optimization 下最基础的优化器节点之一。
- Adam：Adam 可以理解为在 SGD 基础上进一步加入动量估计和自适应步长机制。
- Momentum：Momentum 是对 SGD 的直接增强，目标是降低抖动、提高沿一致方向前进的效率。
- RMSProp：RMSProp 主要改进的是步长缩放方式。
- Learning Rate Schedule：SGD 的效果往往强依赖学习率调度策略。
- Weight Decay：SGD 常与权重衰减联合使用，影响训练稳定性与泛化表现。
