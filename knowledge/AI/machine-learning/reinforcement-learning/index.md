# Reinforcement Learning

## 核心理解

Reinforcement Learning（强化学习）研究的是：智能体如何在环境中通过不断试错，学习一个策略，使长期累计回报最大化。

它和监督学习最大的区别在于，模型不是直接从“标准答案”中学习，而是根据自己采取动作后的结果，接收奖励信号，再逐步调整行为。奖励往往是稀疏的、延迟的、不稳定的，因此强化学习的核心不只是“学会预测”，而是“学会决策”。

## 工作机制 / 构建过程

强化学习通常围绕 agent、environment、state、action、reward 这几个元素展开。

一个基本循环通常是：

1. Agent 观察当前状态 state。
2. Agent 根据策略 policy 选择一个动作 action。
3. Environment 接收动作并发生转移，返回新的状态 next state。
4. Environment 给出即时奖励 reward。
5. Agent 根据这次交互更新策略或价值估计。
6. 重复这一过程，直到一个 episode 结束或持续在线学习。

这个过程的目标不是让某一步奖励最大，而是让长期累计回报更高。因此强化学习常使用 discounted return 来平衡“眼前收益”和“长期收益”。

如果把它抽象成数学问题，最常见的建模方式是 Markov Decision Process，简称 MDP。MDP 用状态转移和奖励函数描述环境，使“如何行动”变成一个优化问题。

## 核心问题

强化学习中的关键难点通常包括：

- Credit Assignment：最终奖励出现得很晚时，如何判断前面哪些动作真正有贡献？
- Exploration vs Exploitation：是继续尝试新动作，还是利用当前已知最优动作？
- Delayed Reward：奖励信号滞后时，学习会变得困难。
- Sample Efficiency：很多方法需要大量交互数据，现实成本较高。
- Stability：训练过程容易震荡、发散或陷入局部最优。
- Generalization：在训练环境有效的策略，未必能迁移到新环境。

## 主要方法

强化学习方法大致可以分成几类：

- Value-Based Methods：学习状态价值或动作价值，再据此选动作，例如 Q-Learning、DQN。
- Policy-Based Methods：直接学习策略分布，输出在某个状态下应采取什么动作。
- Actor-Critic Methods：同时学习策略和价值函数，兼顾优化方向与训练稳定性。
- Model-Free RL：不显式学习环境模型，直接从交互中学习策略或价值。
- Model-Based RL：尝试学习环境动力学，再利用模型做规划或数据增强。
- Offline RL：主要依赖固定历史数据学习，而不是持续在线探索。
- Multi-Agent RL：多个智能体同时交互、合作或竞争的强化学习场景。

## 与其他机器学习范式的区别

监督学习从带标签样本中学习输入到输出的映射，目标通常是降低预测误差。

无监督学习更关注数据结构本身，例如聚类、表示学习和分布建模。

强化学习则关注“行动会影响后续数据分布”的决策问题。模型输出的不只是一个预测结果，而是一个会改变环境轨迹的动作。因此它更适合序列决策、控制、博弈和资源分配问题。

## 典型应用场景

强化学习常见于：

- 游戏 AI：例如 Atari、围棋、星际争霸等复杂决策环境。
- 机器人控制：学习行走、抓取、导航和连续控制策略。
- 推荐与广告：优化长期用户反馈，而不只是单次点击率。
- 资源调度：如缓存、带宽、计算资源分配。
- LLM 对齐：通过人类反馈或偏好信号优化模型行为，例如 RLHF。

## 与相关节点的关系

Reinforcement Learning 是 Machine Learning 的一个重要分支，但它更强调“决策”而不是“静态预测”。

它和 NLP、LLM、Search & Retrieval 不同，关注点不是文本表示或检索排序，而是在交互式环境中优化长期行为。不过在现代 AI 系统中，强化学习也常用于优化推荐策略、Agent 行为和 LLM 对齐流程。
