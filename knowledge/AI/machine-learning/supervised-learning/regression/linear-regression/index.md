# Linear Regression

## 核心理解

Linear regression 是最基础的监督学习模型之一，用来描述输入特征与连续输出之间的线性关系。

它的核心假设是：目标值可以表示为输入特征的线性组合，再加上一个偏置项。

因此，linear regression 既是回归问题的基础方法，也是理解更复杂线性模型和广义线性模型的重要入口。

## 模型形式

在线性回归中，若输入特征向量为 $x = (x_1, x_2, \dots, x_n)$，则模型通常写成：

$$
\hat{y} = w_1x_1 + w_2x_2 + \cdots + w_nx_n + b
$$

也可以写成向量形式：

$$
\hat{y} = w^\top x + b
$$

其中：

- $w$ 是模型参数，也就是各特征的权重。
- $b$ 是偏置项。
- $\hat{y}$ 是模型预测值。

## 它在学什么

linear regression 要学习的是：每个输入特征对输出有多大影响，以及这些影响如何组合成最终预测值。

如果把它看成函数拟合问题，那么它学到的是一条直线、一个平面，或更高维空间中的一个超平面。

因此，所谓“线性”，指的是模型对参数是线性的，预测结果是特征加权求和的结果。

## 为什么叫线性

这里的“线性”首先指模型对参数的线性结构，而不一定意味着问题本身只能处理表面上看起来“直线型”的数据。

例如，如果先对输入做特征变换，再做线性回归，模型仍然是线性模型，但它可以表达更复杂的关系。

所以线性回归的核心不是“世界一定是直线的”，而是“模型输出由一组特征线性组合而成”。

## 训练目标

linear regression 最常见的训练目标是最小化预测值与真实值之间的平方误差，也就是最小化 MSE：

$$
\text{MSE} = \frac{1}{m}\sum_{i=1}^{m}(\hat{y}_i - y_i)^2
$$

这意味着模型会寻找一组参数，使整体预测误差尽可能小。

因此，linear regression 和 [MSE](/Users/bytedance/Documents/knowledge%20database/knowledge/AI/machine-learning/loss-functions/mse/index.md:1) 有非常直接的关系。

## 如何求解

在线性回归中，参数求解常见有两种思路：

- 解析解：直接通过正规方程求得参数。
- 迭代优化：通过梯度下降等方法逐步更新参数。

解析解更适合特征规模不太大时的理论分析和直接求解。

梯度下降更适合数据规模较大、特征较多，或需要和统一训练框架结合时使用。

## 残差的意义

对于单个样本，真实值与预测值之间的差：

$$
y - \hat{y}
$$

通常称为 residual，也就是残差。

残差反映的是模型在该样本上的拟合偏差。线性回归的很多分析，例如误差分布、异常点影响、拟合质量判断，都与残差密切相关。

## 优点

- 形式简单，容易理解和实现。
- 参数可解释性强。
- 训练和推断成本通常较低。
- 是很多更复杂模型的基础出发点。

## 局限

- 只能直接表达线性结构。
- 对异常值比较敏感，尤其在平方误差目标下更明显。
- 如果特征关系复杂、交互强或明显非线性，基础线性回归往往不够。
- 当特征之间高度相关时，参数解释可能变得不稳定。

## 与 Logistic Regression 的区别

名字上虽然都叫 regression，但 linear regression 和 logistic regression 解决的问题不同：

- linear regression 通常用于连续值预测。
- logistic regression 通常用于分类，尤其是二分类。

两者都属于线性模型家族，但输出形式和任务目标不同。

## 与相关节点的关系

- `Regression` 讨论连续值预测这一任务域。
- `MSE` 讨论 linear regression 中常见的训练目标。
- `Optimization` 讨论如何通过解析解或梯度下降求解参数。
- `Generalization` 讨论训练集上拟合得好，是否能在新样本上也表现良好。
