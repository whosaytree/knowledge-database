# BERT

## 核心理解

BERT 是一种基于 Transformer Encoder 的预训练语言模型。

它的重要性不在于“它是一个能直接聊天的大模型”，而在于它定义了一条非常有影响力的技术路线：

- 先在大规模文本上做自监督预训练。
- 再把预训练得到的参数迁移到下游 NLP 任务。
- 用统一的模型骨架适配分类、序列标注、句对判断、问答等任务。

从知识结构上看，BERT 更应该被理解为“预训练语言模型时代里，encoder-only 路线的代表节点”。

## BERT 到底解决了什么问题

在 BERT 之前，很多 NLP 系统往往依赖：

- 静态词向量。
- 任务专用模型。
- 只能单向建模上下文的语言模型。

这些方法的共同问题是：

- 同一个词在不同上下文中的表示不够灵活。
- 不同任务之间难以共享统一表征。
- 模型对句子整体语义和双向上下文的利用有限。

BERT 的核心推进是：

- 用深层双向上下文表示来编码文本。
- 让同一个 token 在不同上下文里有不同表示。
- 通过大规模预训练得到通用语言表征，再迁移到具体任务。

所以它不是简单“把词向量做大”，而是把 NLP 的建模单位从静态词表示推进到上下文化表征。

## 名字里的 Bidirectional 是什么意思

BERT 全称是 `Bidirectional Encoder Representations from Transformers`。

这里的关键是 `Bidirectional`。

它强调的是：模型在编码一个 token 时，不只看左边，也看右边上下文。

例如：

```text
句子：bank of the river
句子：go to the bank to deposit money
```

`bank` 在两个句子里含义不同。

如果表示只看单侧上下文，模型更难稳定地区分这种差异。
BERT 通过双向上下文编码，让 token 表示更依赖整句语义环境。

要注意：

- 这里的“双向”不是说推理时来回生成文本。
- 它指的是 encoder 在表示学习阶段同时吸收左右文信息。
- 这和自回归生成模型的工作方式不同。

## BERT 的基础结构

BERT 建立在 Transformer 的 encoder stack 上。

可以粗略理解为：

```text
input tokens
  -> token embedding + position embedding + segment embedding
  -> multiple Transformer encoder layers
  -> contextualized token representations
```

它没有 decoder，也不负责逐 token 生成文本。

因此：

- BERT 擅长“理解型”任务。
- 不擅长开放式文本生成。
- 它的输出重点是表征，而不是续写。

这也是为什么 BERT 更常出现在分类、抽取、匹配、检索编码等任务里，而不是聊天生成链路里。

## 输入表示由什么组成

BERT 的输入通常由三部分相加得到：

- Token Embedding：词或子词本身的表示。
- Position Embedding：位置信息。
- Segment Embedding：句子 A / 句子 B 区分信息。

输入序列里还会出现一些特殊 token：

- `[CLS]`：常作为整句或整段的聚合表示入口。
- `[SEP]`：分隔句子或片段。
- `[MASK]`：在预训练中用于遮盖部分 token。

例如句对输入常写成：

```text
[CLS] sentence A tokens [SEP] sentence B tokens [SEP]
```

这种格式让模型能统一处理单句任务和句对任务。

## 预训练为什么重要

BERT 的关键不是只有结构，而是“Transformer Encoder + 大规模自监督预训练”这套组合。

预训练的意义在于：

- 先让模型在海量文本里学语言统计规律和上下文依赖。
- 再把这种通用能力迁移到具体任务。
- 这样下游任务不需要从零开始学语义表示。

这套范式后来几乎影响了整个 NLP 领域。

可以把它理解成：

```text
通用语言规律 -> 预训练参数
具体任务数据 -> 微调适配
```

因此 BERT 成功的核心之一，是它证明了“预训练 + 微调”在 NLP 中可以形成统一方法论。

## Masked Language Modeling

BERT 最经典的预训练目标是 `Masked Language Modeling`，简称 `MLM`。

基本思路是：

- 在输入句子里随机遮掉一部分 token。
- 模型根据其余上下文去预测被遮掉的位置。

例如：

```text
The capital of France is [MASK].
```

模型需要根据上下文预测 `Paris`。

这个任务的重点不是背答案，而是逼模型学习：

- 词和词之间的上下文依赖。
- 句法与语义结构。
- 局部信息和全局上下文之间的关系。

它之所以适合 BERT，是因为 BERT 的 encoder 能同时看左右上下文，因此更适合做“补全被遮住位置”的表示学习。

## Next Sentence Prediction 在做什么

原始 BERT 还引入了 `Next Sentence Prediction`，简称 `NSP`。

它的目标是让模型判断：

- 句子 B 是否是句子 A 在原文中的下一句。

这个目标最初是为了增强句间关系建模能力，方便迁移到：

- 句对分类
- 自然语言推断
- 问答
- 段落级理解

后来很多后续模型对 NSP 的价值提出质疑，有的直接移除，有的换成别的训练目标。
所以从今天回看，NSP 不是 BERT 最持久的贡献，但它是理解原始 BERT 训练设计时必须知道的一部分。

## BERT 输出怎么用于下游任务

BERT 经过预训练后，可以通过微调接到很多任务头上。

典型方式包括：

### 1. 句子级分类

用 `[CLS]` 对应的输出表示整句，再接分类层：

```text
[CLS] representation -> classifier -> label
```

适用于：

- 情感分类
- 主题分类
- 意图识别

### 2. Token 级任务

直接使用每个 token 的上下文化表示：

- 命名实体识别
- 词性标注
- 序列标注

### 3. 句对匹配

把两个句子拼接输入，再做分类或匹配判断：

- 自然语言推断
- 语义匹配
- 问答候选判断

### 4. 抽取式问答

让模型预测答案 span 的起点和终点位置。

这类任务非常符合 BERT 的 encoder 表征能力，因为答案通常是从上下文中抽取，而不是生成。

## 为什么说它是理解型模型

BERT 的强项来自 encoder-only 结构：

- 输入全部上下文后再编码。
- 每个 token 表示都依赖整段输入。
- 重点在语义理解和表征提取。

这使它适合：

- 分类
- 抽取
- 匹配
- 编码
- reranking
- embedding 基础模型

但不适合：

- 长文本自由生成
- 自回归续写
- 对话式生成

所以不能把 BERT 和 GPT 直接当作同一类模型去理解。
它们都属于预训练语言模型，但路线不同：

- BERT 是 encoder-only，偏理解。
- GPT 是 decoder-only，偏生成。

## 实现方法与工程细节

### 1. 子词分词很关键

BERT 通常不是按完整单词编码，而是按 subword 单位编码。

这样做的好处是：

- 缓解未登录词问题。
- 控制词表规模。
- 让模型更容易泛化到罕见词、复合词和词形变化。

### 2. 最大长度限制直接影响任务设计

BERT 有固定最大输入长度。

这会影响：

- 长文档切分策略。
- 问答窗口滑动方式。
- 检索和编码任务里的 chunking 设计。

如果输入超过上限，就必须截断或切片，这在工程上是非常实际的约束。

### 3. 微调成本低于从零训练

BERT 的真正工程价值之一，是下游任务只需：

- 接一个轻量任务头。
- 用相对小得多的标注数据训练。
- 在预训练参数上继续优化。

这比从零训练任务模型高效得多，也推动了“预训练模型复用”成为默认范式。

### 4. 迁移时要注意预训练任务和下游任务的错位

虽然 BERT 很通用，但不是任何任务都能无损迁移。

例如：

- 对生成式任务并不自然。
- 对超长上下文任务会受长度约束。
- 对需要极强领域术语的任务，可能需要领域继续预训练或专门微调。

## BERT 带来的方法论影响

BERT 的影响不只是一篇模型论文，而是几件事同时成立：

- Transformer Encoder 可以成为统一语言理解骨架。
- 自监督预训练可以显著提升下游任务效果。
- NLP 可以围绕“通用预训练模型 + 任务微调”组织工程体系。
- 模型家族会围绕同一路线持续变体化和增强。

后续很多模型其实都可以看成是在沿着这条线继续推进：

- 改预训练目标。
- 改训练数据。
- 改参数效率。
- 改层数规模。
- 改注意力和表征方式。

所以 BERT 更像一个基础分水岭节点，而不是孤立模型名词。

## 与相关节点的关系

BERT 应位于 `NLP -> Transformer` 路径下。

它和相关方向的关系大致是：

- 它建立在 Transformer Encoder 之上。
- 它属于预训练语言模型，但不是生成式 LLM。
- 它和 GPT 代表了不同的预训练路线。
- 它可以作为很多检索、匹配、rerank、分类模型的基础编码器。
- 后续很多 encoder-only 变体都可以视为对 BERT 路线的延伸。
