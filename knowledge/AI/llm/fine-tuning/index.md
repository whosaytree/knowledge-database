# Fine-tuning

## 核心理解

Fine-tuning（微调）是在已有预训练模型的基础上，继续使用特定数据训练模型，使模型更适合某类任务、领域、风格或交互方式。

在 LLM 中，微调通常不从零开始训练模型，而是利用基础模型已经获得的语言能力、知识表示和推理模式，再通过额外训练改变模型的行为分布。它常用于让模型更会遵循指令、适应专业领域、稳定输出格式，或学习某类应用场景中的回答方式。

## 工作机制 / 构建过程

Fine-tuning 的基本过程通常包括：

1. 选择基础模型：根据能力、参数规模、上下文长度、许可证和部署条件选择可微调模型。
2. 准备训练数据：构造任务样本、领域文本、问答对、指令数据或偏好数据。
3. 数据清洗与格式化：去除低质量样本，统一 prompt、response、system message、标签和输出格式。
4. 设定训练目标：常见目标是让模型在给定输入后生成目标输出，并通过损失函数优化预测结果。
5. 执行训练：在基础模型参数或新增适配参数上继续训练。
6. 评估效果：用验证集、人工检查、自动指标和线上任务表现判断微调是否有效。
7. 部署与迭代：将微调后的模型用于推理服务，并根据反馈继续修正数据和训练配置。

在训练时，输入样本会被转换为 token 序列，模型根据上下文预测目标 token。训练过程通过反向传播更新参数，使模型在相似输入下更倾向于生成期望输出。

## 实现方法与工程细节

Fine-tuning 的实现方式通常包括以下几类：

- Full Fine-tuning：更新模型全部参数，效果上限较高，但显存、算力和数据质量要求也高。
- Parameter-Efficient Fine-tuning：只训练少量新增参数或部分参数，用较低成本改变模型行为。
- LoRA：在部分线性层中加入低秩矩阵，只训练这些低秩参数。
- QLoRA：在量化基础模型的同时使用 LoRA 训练，进一步降低显存占用。
- Instruction Tuning：使用指令-回答数据训练模型，使其更好地遵循用户任务。
- Domain Fine-tuning：使用专业领域数据训练模型，使其更熟悉特定术语、格式和知识分布。
- Supervised Fine-tuning：使用输入与目标输出成对的数据进行监督训练，简称 SFT。

训练时需要重点关注：

- Learning Rate：学习率过高容易破坏原有能力，过低则可能学习不足。
- Batch Size：影响训练稳定性、显存占用和收敛表现。
- Epoch：训练轮数过多可能过拟合，过少可能效果不明显。
- Data Quality：微调通常对数据质量非常敏感，少量高质量数据可能优于大量噪声数据。
- Evaluation Set：需要保留未参与训练的数据，用于判断模型是否真正泛化。
- Catastrophic Forgetting：模型在学习新任务时可能削弱原有能力。
- Overfitting：模型可能记住训练样本，而没有学到稳定行为模式。

## 与其他 LLM 技术的关系

Fine-tuning 与预训练、提示词工程、RAG 和对齐关系密切。

预训练提供基础能力，Fine-tuning 在此基础上调整模型行为。提示词工程通过输入控制模型输出，不改变模型参数。RAG 通过外部检索补充上下文，适合需要动态知识或可追溯资料的任务。对齐方法则更关注模型输出是否符合人类偏好、安全要求和交互规范。

实践中，Fine-tuning 常和 RAG、提示词工程一起使用。微调用于固定模型的行为模式和输出风格，RAG 用于提供外部知识，提示词用于表达当前任务约束。
