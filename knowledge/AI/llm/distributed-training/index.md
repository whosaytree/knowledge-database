# Distributed Training

## 核心理解

Distributed Training 讨论的是：当单卡显存、单机算力或训练吞吐不足时，如何把模型训练过程拆到多张卡、多个进程甚至多台机器上执行。

在 LLM 训练里，分布式训练不是单一技术，而是一组并行与状态管理策略的组合。它要解决的核心问题包括：模型参数放在哪里、梯度如何同步、优化器状态如何保存、激活值如何传递，以及通信开销如何控制。

## 关键问题

- 参数、梯度和优化器状态如何分布在多卡之间？
- 前向与反向传播时，哪些张量需要通信？
- 显存瓶颈来自参数、激活值还是优化器状态？
- 吞吐量提升是否被通信延迟抵消？
- 训练复杂度和故障恢复成本是否可控？

## 主要方向

- [FSDP](fsdp/index.md)：按模块分片参数、梯度和优化器状态的全分片数据并行方法。
- [Megatron](megatron/index.md)：面向大模型训练的多维并行框架与工程体系。
- DDP：每张卡保留完整模型副本，通过梯度同步实现数据并行。
- ZeRO：按阶段分片优化器状态、梯度和参数的内存优化方案。
- Tensor Parallelism：把单层计算切分到多卡上执行。
- Pipeline Parallelism：把模型不同层切到不同设备上顺序执行。

## 与 LLM 的关系

LLM 的参数规模、序列长度和训练 batch 往往很大，单卡训练很快会碰到显存和吞吐瓶颈。因此分布式训练几乎是大模型预训练与大规模微调的基础设施层能力。

它和模型结构、优化器、混合精度、checkpoint、数据加载、通信拓扑都有强耦合关系。
